**Técnica de refactorización aplicada en cada caso**


**1\. Claridad y Comprensión**


Sustituir nombres poco claros por nombres descriptivos: Se logra mediante **Rename Method (Renombrar Método) o Rename Variable**. Es la base para mejorar la comprensión del código. 
**Mejorar la legibilidad:** Se aplican técnicas como **Extract Variable** para simplificar expresiones complejas y hacer que el código "se explique solo".


**2\. Estructura y Responsabilidad de Métodos**


**Extracción de métodos:** La técnica es **Extract Method**. Se utiliza para dividir bloques de código largos y organizar el código en partes más pequeñas. 
Crear métodos con una única responsabilidad: Se apoya en **Extract Method y Replace Method with Method Object**, evitando que toda la lógica esté concentrada en un único lugar. 
**División de métodos largos:** Además de extraer métodos, se puede usar **Decompose Conditional** para separar la lógica de comprobación del proceso principal.


**3\. Gestión de Código Duplicado**


Detectar código repetido y eliminación de código duplicado: La técnica principal es **Extract Method (para mover la lógica común a un solo lugar)**


**4\. Complejidad Lógica**


**Simplificación de condiciones:** Se utilizan técnicas como **Consolidate Conditional Expression** (unir varias condiciones que llevan al mismo resultado) o **Replace Nested Conditional with Guard Clauses**. 
**Reducir la complejidad de estructuras condicionales:** Una técnica avanzada es reemplazar lo condicional por el polimorfismo, lo que permite que el comportamiento varíe según el tipo de objeto sin usar múltiples if o switch.


**Justificación de cada mejora**


1. **Renombrado** 
  Hemos renombrado las variables n,p,c y p por nombre, precio, cantidad y producto respectivamente para facilitar su legibilidad. 
   
2. **Documentación** 
  Hemos mejorado la comprensión del código añadiendo comentarios a cada fragmento del mismo explicando las funcionalidades de los fragmentos para su mejor comprensión.


3. **Extracción de métodos** 
  De todo el método main hemos extraído las funciones cargarProductosCarrito(), procesarPedidoCompleto(List\<Producto\> productos), calcularSubtotalConDescuento(Producto p), calcularGastosEnvio(double subtotal), imprimirFichaProducto(Producto p, double subtotal, double envio) e imprimirResumenFinal(double total) 
   
4. **Dividir bloques de código largos** 
  Hemos fragmentado el código separándolo en dos archivos llamados Producto.java para el constructor de los productos y Tienda.java para la lógica de negocio e interfaz , creando funciones en Tienda.java para dividir el gran bloque de código inicial en métodos independientes e igual de funcionales. 
   
5. **Eliminación de código duplicado** 
  Hemos eliminado partes de código que nos han resultado repetitivas y que se pueden simplificar para que sea el código más eficiente. 
   
6. **Simplificación de condiciones** 
  Simplificamos condiciones para que sean más entendibles y eficientes. 
   
   
7. **Reducir la complejidad de estructuras condicionales** 
  Hicimos que los `if` sean más simples, claros y fáciles de seguir, evitando lógica liosa o difícil de leer. 
  En el código original, la complejidad venía de cosas como condiciones mezcladas con cálculos, varias decisiones seguidas dentro del mismo bloque y lógica metida directamente en el `main`
